# AXIS_MOTION_GUIDELINES.md  
AXIS_ROOT v10.5 — Движение, Физика и Микровзаимодействия

Этот документ описывает фундаментальную философию движения в AXIS OS.  
Анимация — это не украшение.  
Это язык интерфейса, форма коммуникации и инструмент управления вниманием.

Хорошая анимация увеличивает конверсию.  
Плохая — убивает UX, батарею и скорость.

Эти правила используются всеми модулями AXIS:  
Designer, Creative Engine, Developer, Knowledge Syncer, QA Engine.

────────────────────────────────────────
# 1. ФИЛОСОФИЯ ДВИЖЕНИЯ AXIS
────────────────────────────────────────

1) **Motion = Meaning**  
Каждое движение должно что-то объяснять пользователю:  
переход, появление, акцент, подтверждение, ошибку.

2) **Physics-Based UI**  
Интерфейс должен ощущаться «живым», но не мультяшным.  
Пример: легкая инерция, смягчённые касания, физическая логика.

3) **Скорость = Уважение к пользователю**  
Долгие анимации — воруют время.  
Цель AXIS: максимальная скорость при максимальной ясности.

4) **Energy-Safe Motion**  
Мобильные устройства = ограничения CPU/GPU.  
Анимации должны быть энергоэффективными.

5) **Motion as Accessibility**  
Анимация должна помогать ориентироваться, а не мешать.

────────────────────────────────────────
# 2. ПРАВИЛА ПРОИЗВОДИТЕЛЬНОСТИ
────────────────────────────────────────

### Разрешено:
- transform (translate, scale, rotate)
- opacity
- filter (только при низкой интенсивности)
- CSS variables for timing

### Запрещено:
- layout thrashing (top/left)
- box-shadow heavy transitions
- background-position animations
- JS-анимации без причины
- анимации более 300ms

### Golden Rule AXIS:


Если анимация снижает LCP/INP — она удаляется.


────────────────────────────────────────
# 3. СКОРОСТЬ И ТАЙМИНГ
────────────────────────────────────────

### Базовые параметры:
- **duration_primary:** 160–220ms  
- **duration_secondary:** 120–160ms  
- **duration_micro:** 80–120ms  

### Типы ускорений:
- **ease-out:** для появления  
- **ease-in:** для исчезновения  
- **ease-in-out:** для движения внутри блока  
- **cubic-bezier(0.22, 1, 0.36, 1):** «элегантная подпружиненность», стандарт AXIS  

────────────────────────────────────────
# 4. ПРИНЦИПЫ ПСИХОФИЗИКИ
────────────────────────────────────────

### Pre-Attentive Motion
Глаза замечают движение за ~0.15с.  
Мы используем motion для акцентов:  
- hover активация  
- изменение цены  
- переход состояний

### Human Perception Rule  
Пользователь считает анимацию «мгновенной», если:


< 200ms


### Movement Hierarchy
В AXIS есть уровни внимания:

1) **Primary Motion** — крупные события (открытие меню, смена секции)  
2) **Secondary Motion** — небольшие переходы (открытие блока фильтра)  
3) **Micro Interactions** — tiny эффекты (клик, hover, ripple)

────────────────────────────────────────
# 5. MICRO-INTERACTIONS (МИКРОВЗАИМОДЕЙСТВИЯ)
────────────────────────────────────────

### Примеры:
- Ripple на CTA кнопке  
- Product card hover lift (2–4px)  
- Icon wiggle (error state)  
- Add-to-cart pulse  
- Tab switching indicator slide  

### Правила:
- не отвлекать  
- не раздражать  
- быть быстрым  
- быть полезным

────────────────────────────────────────
# 7. ACCESSIBILITY MOTION RULES
────────────────────────────────────────

### Уважать prefers-reduced-motion
Если у пользователя включён режим снижения анимаций:

- отключить все вторичные анимации  
- заменить пружины на fade  
- заменить hover-motion на static  
- удалить параллакс  

### Запрещено:
- быстрые flashing transition  
- motion loops без конца  
- анимации, пересекающие весь экран  

────────────────────────────────────────
# 8. ПРОТОКОЛ ДЛЯ AI-GENERATED MOTION
────────────────────────────────────────

AI обязан:

1. Проектировать анимацию, учитывая нишу.  
2. Проверять duration < 220ms.  
3. Проверять доступность.  
4. Проверять энергоэффективность.  
5. Генерировать только transform + opacity.  
6. Интегрировать tokens из design_tokens.json.
