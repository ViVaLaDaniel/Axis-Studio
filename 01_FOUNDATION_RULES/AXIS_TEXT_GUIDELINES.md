# AXIS_TEXT_GUIDELINES.md  
AXIS_ROOT v10.5 — Психолингвистические правила UX-текста, Copywriting и CTA

Этот документ — фундаментальная текстовая конституция AXIS OS.  
Он определяет поведение текста в интерфейсе:  
как писать, что писать, зачем писать и как текст влияет на продажи.

Сильный UX-текст = ↑ конверсия, ↓ ошибки, ↑ доверие.

────────────────────────────────────────
# 1. ОСНОВНАЯ ФИЛОСОФИЯ ТЕКСТА В AXIS
────────────────────────────────────────

1) **Ясность > Креативность**  
   Пользователь должен понять действие за 0.4 секунды.

2) **Один экран — одна мысль**  
   Не нагружай человека конкурирующими сообщениями.

3) **Каждый текст должен иметь цель**  
   CTA = действие  
   Подзаголовок = раскрытие  
   Описание = ориентация  
   Формы = уверенность  

4) **Текст — это часть UX, а не украшение**

5) **Тон бренда = стиль текста = эмоциональный опыт**

────────────────────────────────────────
# 2. CTA-ПСИХОЛОГИЯ (CALL TO ACTION)
────────────────────────────────────────

CTA должен соответствовать трем принципам:

## A) Действие  
Использовать глаголы: Купить, Смотреть, Выбрать, Добавить.

## B) Снижение страха  
Убирать риски через microcopy:  
«Без скрытых платежей»  
«Можно отменить в любое время»

## C) Уверенность  
«Гарантия возврата»  
«Сертифицированный продукт»  
«3,000+ отзывов покупателей»

### Примеры идеальных CTA:
- «Добавить в корзину»  
- «Быстрый просмотр»  
- «Оформить заказ безопасно»  
- «Выбрать размер»  
- «Создать комплект со скидкой»

────────────────────────────────────────
# 3. MICROCOPY (МИКРО-ТЕКСТЫ)
────────────────────────────────────────

Microcopy = мини-тексты, которые спасают UX.

Используется в:
- формах
- ошибках
- подсказках
- вариациях товара
- фильтрах
- чек-ауте

## Правила:
- Никогда не обвиняй пользователя  
- Дай короткое, конкретное действие  
- Не создавай напряжения  
- Дай контекст:

Примеры:
- «Введите корректный email» (вместо «Неверный email»)  
- «Товар заканчивается!»  
- «Осталось 2 на складе»  
- «Бесплатная доставка от 60$»  

────────────────────────────────────────
# 4. ЧИТАБЕЛЬНОСТЬ (READABILITY)
────────────────────────────────────────

UX-текст должен соответствовать:

- Flesch-Kincaid Grade: 6–8  
- Среднее предложение: 8–12 слов  
- Один абзац = 1 мысль  
- Максимум активных глаголов  
- Минимум существительных подряд  

Rules:
- Избегать сложных конструкций  
- Не использовать «канцелярит»  
- Не писать длинные заголовки (до 45 символов)  
- Структурировать текст визуально, если он длинный  

────────────────────────────────────────
# 5. ТЕКСТ ДЛЯ РАЗНЫХ НИШ
────────────────────────────────────────

## Luxury
- меньше текста  
- высокий контраст  
- уверенный тон  
- избегать скидочных слов  

Пример:
«Создано, чтобы впечатлять.»

## Beauty
- мягкий тон  
- забота, деликатность  
- эмоции  

Пример:
«Нежный уход, вдохновлённый природой.»

## Fitness
- энергия  
- мотивация  
- действие  

Пример:
«Твой следующий шаг начинается здесь.»

## Tech
- точность  
- факты  
- доверие  

Пример:
«Мощность, которой можно доверять.»

────────────────────────────────────────
# 6. ПОВЕДЕНЧЕСКИЕ ПРИНЦИПЫ (CXL, NNGroup)
────────────────────────────────────────

## Loss Aversion
Фразы:  
«Не пропустите скидку»  
«Осталось мало»

## Social Proof
«20000+ довольных клиентов»

## Anchoring
«Ранее 149$ — сейчас 99$»

## Cognitive Fluency
Текст должен быть:
- простым  
- коротким  
- предсказуемым  

────────────────────────────────────────
# 7. ГОЛОС & ТОН
────────────────────────────────────────

## UX-голос AXIS:
- уверенный  
- точный  
- честный  
- без лишней рекламы  
- без давления  

## Тон:
- зависит от ниши (см. раздел 5)  
- без сарказма  
- без агрессии  
- ясный и теплый  

────────────────────────────────────────
# 8. ТЕКСТОВЫЕ ОШИБКИ, КОТОРЫХ НИКОГДА НЕЛЬЗЯ ДОПУСКАТЬ
────────────────────────────────────────

- пустые CTA  
- слишком много слов  
- формулировки без действия  
- пассивный залог  
- обман / давление  
- сложные предложения  
- противоречивые сообщения  
- заголовки без смысла  

────────────────────────────────────────
# 9. AUTO-GENERATION ПРОТОКОЛ ДЛЯ AI
────────────────────────────────────────

AI должен:

1. Генерировать текст с учётом ниши и архетипа.  
2. Проверять читабельность автоматически.  
3. Сравнивать с психолингвистическими правилами.  
4. Гарантировать CTA clarity.  
5. Предотвращать перегрузку информацией.
